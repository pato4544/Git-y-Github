Clase 3: Comandos Basicos de Git (init, add, commit)

  1. Inicializar un Repositorio Local

        2. Definición:

      • Un repositorio es un espacio donde se guarda el código y su historial de cambios.

        Comando:

      • Para inicializar un nuevo repositorio, utilizamos el comando: [git init]
        (este comando crea un nuevo directorio .git en la carpeta, que contiene todos los archivos necesarios para el seguimiento del proyecto).

        Demostración:

      1. Abrir la terminal y ejecutar mkdir mi_proyecto para crear un nuevo directorio.

      2. Navegar a la carpeta con cd mi_proyecto.

      3. Ejecutar git init para inicializar el repositorio.

        Recursos:
      • ¿Como inicializo un repositorio en git?

  2. Añadir Cambios al Área de Preparación

        3. Definición:

      • El área de preparación (staging area) es un lugar donde se almacenan los cambios que se preparan para ser confirmados.

        Comando:

      • Para añadir un archivo al área de preparación, se usa: [git add nombre_del_archivo]

      • Para añadir todos los archivos nuevos y modificados: [git add .]

        Demostración:

      1. Crear un archivo de texto en el directorio del proyecto: echo "Hola, Git!" > archivo.txt.

      2. Usar git add archivo.txt para añadirlo al área de preparación.

        Recursos:
      • ¿Que es el Staging Area?

  3. Confirmar Cambios

        4. Definición:

      • Un commit es una instantánea de los archivos en el área de preparación, junto con un mensaje descriptivo.

        Comando:

      • Para confirmar cambios, se usa: [git commit -m "Mensaje descriptivo del commit"]

        Demostración:

      1. Para confirmar cambios, se usa: git commit -m "Añadir archivos de texto con saludo"
